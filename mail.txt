Быстренько накидал back для демо. В дальнейшем с переработками можно использовать в основной версии.

Back представляет из себя Rest API без логики работы(пока неизвестно какая логика нужна, просто инфраструктура для frontend). Структуру и описание можно посмотреть при помощи swagger (), есть описание которое старалось соответствовать чуть менее чем польностью
реализации. Думаю из описания будет понятно что да как. Демо Rest API представляет собой самодостаточный Application, настроен на порт 9090 (можно изменить в настройках),
использует внутреннюю in-memory СУБД (не нужна внешняя СУБД, для демо самое оно). При запуске заполняется необходимая для демо информация. А именно:

Новости: 
Добавлены три новости, не очень информативные. Можно изменить на что-нибудь нормальное.

Доступ в API, по ключевым методам (в описании есть необходимая информация), осуществялется при помощи token (они разделены на web токены и device токены, именно посредствам токенов наделяются права
на управления теми или иными областями). Грубо сказать, методы API разделены на 2 части: первая часть - общедоступные, любой незарегистрированный пользователь может доступиться до определенных методов и 
вторая - доступ по правам (управление новостями, настройкми, ошибками и т.д.). Токены созданные для зарегистрированных пользователей в демо, можно посмотреть в логах (в аттаче скрин, где есть выделенные 
области изображения).

Необходимые демо эелемнты добавляются в ru.phi.modules.demo.DemoConfiguration в модуля rest-demo.
